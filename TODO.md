# TO-DO Items

*Note:* These items are ideas for future enhancements. Their suitability needs
to be assessed as part of the initial work, and inclusion here does not 
represent a roadmap or any guarantee that any features will actually be
implemented.

## Navigation

- Consider options for alternate views of servers/clusters, e.g. tagging, or
    geographically based.
- Consider adding "database" as a new lowest-level scope for dashboards, 
    alerts, probes, and blackouts etc.

## Dashboards

- Add support for customisable dashboards.
- Add a general reporting tool, allowing charts and graphs to be generated
    from collected metrics, with scheduled execution.

## Diagnostics

- Add support for wait state monitoring.
- Add a server log browser.

## Scaling

- Add support for distributing load over multiple collectors and alerters, 
    with HA.

## General

- Add an audit log to record potentially destructive and security related
    actions etc.
